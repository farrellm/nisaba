## 2024-05-24 - SSRF Bypass via Encoded Dot Segments
**Vulnerability:** A Server-Side Request Forgery (SSRF) bypass was possible in the Reddit API integration (`redditPostPath`). Although literal dot segments (`.`, `..`) were correctly rejected, an attacker could supply URL-encoded sequences (`%2e`, `%252e`, `%2f`) which successfully circumvented standard splitting based on literal `/`. Once forwarded, the upstream server could decode and normalize these values, resulting in an out-of-bounds path traversal against the `oauth.reddit.com` endpoints.
**Learning:** Checking for path traversals by splitting on `.` or `..` is insufficient when downstream APIs accept encoded input. Additionally, Go's `url.URL` parser does not fully decode paths with dot-segments in `parsed.Path` if encoded elements like `%2f` are present, which are preserved in `parsed.RawPath`.
**Prevention:** In Go `url.URL` validation, proactively check both `parsed.Path` and `parsed.RawPath` (if present) for encoded dangerous characters such as `%2e` and `%2f` in paths constructed from user input to proxy requests. Use `strings.ToLower` on the raw path before looking for these encoded combinations as an effective defense-in-depth tactic for strict allowlists.

## 2024-05-24 - Missing Rate Limiting on Sensitive Endpoints
**Vulnerability:** Missing rate limiting on authentication endpoints (`/login` and `/register`).
**Learning:** Adding a basic rate limit based on IPs can prevent brute force attacks and other automated abuse that targets sensitive endpoints. Using a token bucket rate limiter (e.g., `golang.org/x/time/rate`) inside a middleware scoped to specific routes creates an efficient and safe security layer. When building this middleware, keeping the visitors map scoped to the `RateLimit` closure rather than global prevents rate limit collisions across different endpoints that may use different limits. It's also important to use `net.SplitHostPort(r.RemoteAddr)` for reliable parsing of IPv6 addresses along with IPv4.
**Prevention:** Consider implementing IP-based or user-based rate limiting on sensitive routes using middleware and token buckets. Ensure that state like visitor rate limiters are encapsulated properly within the closure of the middleware, and correctly handle port stripping for client IP addresses.
