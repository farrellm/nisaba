# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Common Commands

```sh
# Database
make db              # start Postgres container
make migrate         # run pending migrations
make migrate-down    # roll back last migration
make db-clean        # wipe container and data volume

# Backend
make backend         # go run ./cmd/server (port 8080)
make backend-watch   # like backend, auto-restarts on changes (needs wgo)
make backend-build   # compile to backend/bin/server
make backend-test    # go test ./...

# Frontend
make frontend-install  # npm install (first time only)
make frontend          # vite dev server (port 5173)
make frontend-build    # production build to frontend/dist/ (runs tsc first — also the typecheck)
```

Quick checks: `gofmt -l backend/` (format), `cd frontend && npx tsc --noEmit` (typecheck only). There is no separate frontend lint step. There is also no frontend test runner — verify pure-logic utilities (e.g. `frontend/src/lib/`) with a quick `node -e` port of the function. The frontend tsconfig targets ES2020 (`lib: ES2020`), so newer stdlib like `String.replaceAll`/`Array.at` isn't typed — use ES2020-compatible equivalents.

Install golang-migrate before running migrations:
```sh
go install -tags 'postgres' github.com/golang-migrate/migrate/v4/cmd/migrate@latest
```

## Core Concept: Modes & Blocks

Nisaba is a tool for **writing with LLMs**. A **document** holds an ordered list of **blocks** plus its own key/value **attributes** (a shared namespace) and a **selected model**.

There is a **fixed, code-defined set of modes** (`backend/internal/mode`). Each mode declares a fixed set of input **keys**, an **output** key, a **mustache template**, and an optional set of **tools** (`Tools []llm.Tool`, `json:"-"`). The set is fixed at build time — there is no runtime CRUD.

The lifecycle:
1. **Add block** — the user picks a mode. The new block's attributes are seeded from the document's attributes for that mode's keys (empty string where the document has no value).
2. **Edit** — the user edits the block's key/values.
3. **Run** — the mode's mustache template is rendered against the block's key/values to produce a **prompt**; the prompt is sent to the document's selected model; the **response** is appended to the block's responses. The response is then processed into the document's attributes two ways (both apply): every **top-level XML tag** becomes an attribute (tag name → key, inner text → value; nested tags stay verbatim in the value), and when the mode declares a non-empty `output` key the full response is also saved under it. The `output` key is optional.

**The LLM call is provider-agnostic.** `RunBlock` (`internal/handler/block.go`) renders the prompt and the system prompt (`mode.SystemPrompt(username)`, a per-user-overridable mustache template) and calls `llm.Generate(ctx, model, system, prompt, tools)` (`internal/llm`), a thin wrapper over the [GoAI SDK](https://github.com/zendev-sh/goai) that routes each model through the provider named in its `Model.Provider` — its own vendor (Anthropic/OpenAI/Google) or the OpenRouter aggregator — GoAI reads that provider's key from the environment (`ANTHROPIC_API_KEY`, `OPENAI_API_KEY`, `GEMINI_API_KEY`/`GOOGLE_GENERATIVE_AI_API_KEY`, `OPENROUTER_API_KEY`, `DEEPSEEK_API_KEY`). `RunBlock` passes the mode's `Tools`; when non-empty, `Generate` attaches each tool and runs GoAI's agentic loop (`WithMaxSteps`, bounded by `maxToolIterations`) so the model can invoke tools and have results fed back, otherwise it does a single generation. Each document picks a model from a single **fixed, cross-provider list** (`llm.Models()`, served at `GET /api/models`); `Run` 400s if no model is selected. Keep all vendor-specific code behind `internal/llm` — never import a single-vendor SDK directly. `llm.Tool` is an alias for GoAI's tool type; build one with `goai.NewTool` (schema auto-generated from a typed input struct's `json`/`jsonschema` tags — no hand-written param schema). `internal/llm/names.go` ships a ready-made `llm.GenerateNameTool` (`generate_name`, returns a non-blocklisted character name) as a worked example — currently attached to the `brainstorm-tools-*` modes. `internal/llm/labels.go` is a second worked example: `llm.SuggestLabels(ctx, story)` hard-codes a model, renders the embedded `templates/suggest-labels.mustache`, and parses `<label>` tags out of the reply (templates that drive a hard-coded-model helper live under `internal/llm/templates/`, since `go:embed` can't reach a parent dir). The same file's `llm.SelectLabels(ctx, story, available)` renders `templates/available-labels.mustache` and filters the reply down to the supplied label set (picking relevant ones from an existing pool rather than inventing new labels).

## Architecture

The app is split into three independent directories: `frontend/`, `backend/`, and `db/`.

**Frontend** (`frontend/`) — Vite + React 18 + TypeScript + MUI v6. The Vite dev server proxies all `/api/*` requests to `http://localhost:8080`, so the browser never makes a cross-origin request during development. Production builds output to `frontend/dist/`.
Routing uses `react-router-dom` (routes in `src/App.tsx`, providers in `src/main.tsx`). Call the API via `src/api/client.ts` (`api.get/post/put`, sends the session cookie) rather than raw `fetch`. Frontend API types are in `src/api/types.ts` (`Document`, `DocumentDetail`, `Block`, `Response`, `Mode`, `LLMModel`); treat any array from the API as possibly `null` and guard with `?? []`. Current user comes from `useAuth()` (`src/auth/AuthContext.tsx`); wrap protected routes in `RequireAuth`, which redirects to `/login`. The document view (`pages/DocumentPage.tsx`) renders blocks via `components/BlockCard.tsx`, adds them via `components/AddBlockDialog.tsx`, and picks the model via the fixed lower-left `components/ModelSelector.tsx` (auto-saves through `PUT /api/documents/{id}`). There is no shared layout wrapper — each page renders its own chrome: `components/Masthead.tsx` for the standard nav, `components/AuthLayout.tsx` (or simply omitting the masthead) for chrome-free pages, and the tab title via the `usePageTitle` hook. Render markdown/LLM output through the reusable `components/Markdown.tsx` (react-markdown + remark-gfm), which carries the editorial styling — don't reach for another markdown lib. It renders no raw HTML (no `rehype-raw`), so XML/HTML tags in model output are dropped unless escaped (`<` → `\<` makes them render as literal text). Per-attribute icon links (in `BlockCard`/`components/DocumentAttributes.tsx`) open chrome-free, full-page standalone views in a new tab (`pages/AttributePage.tsx`, `pages/BlockAttributeDiffPage.tsx`, routed under `/documents/:id/...`) that re-fetch the document by id rather than taking props — mirror this for new attribute viewers. Word-level diffing lives in `lib/wordDiff.ts` (algorithm documented in `lib/wordDiff.md`). Labels UI: `pages/LabelsPage.tsx`, `components/EditLabelsDialog.tsx`, and `components/AuthorField.tsx`.

**Backend** (`backend/`) — Go module `github.com/farrellm/nisaba`. Entry point is `cmd/server/main.go`. Internal packages:
- `internal/config` — reads `ADDR`, `DATABASE_URL`, `CORS_ORIGINS`, `MODE_TEMPLATES_DIR` from env with local dev defaults. Reddit integration reads `REDDIT_CLIENT_ID`/`REDDIT_CLIENT_SECRET` for reading and `REDDIT_USERNAME`/`REDDIT_PASSWORD` for submitting (handlers 503 when the relevant pair is unset)
- `internal/auth` — cookie-session helper (gorilla/sessions, signed, HttpOnly). `SESSION_SECRET` signs the cookie (dev default; prod must override); `SESSION_SECURE=true` sets the Secure flag for HTTPS
- `internal/db` — opens a `pgxpool.Pool` and pings on startup to fail fast
- `internal/handler` — `http.HandlerFunc` closures; data-access handlers take `*store.Store` (built via `store.New(pool)` in main.go), auth-aware ones also take `*auth.Sessions`. Only `Health` still takes the raw pool. Auth flow lives in `auth.go`: `/api/auth/{register,login,logout,me}`, bcrypt-hashed passwords, generic 401 on bad login, 409 on duplicate username. Document CRUD lives in `document.go`: `/api/documents` (list/create) and `/api/documents/{id}` (get/`PUT`/`DELETE`); list takes `?archived=true`. `PUT /api/documents/{id}` updates the selected model, document attribute values, archive state, and/or labels — body fields (`selectedModel`, `attributes`, `isArchived`, `labels`) are optional pointers, each applied only when present (attributes via `MergeDocumentAttributes`, so absent keys survive; labels via `SetDocumentLabels`, which reconciles the join by name, reusing existing labels). Block flow lives in `block.go`: `POST /api/documents/{id}/blocks` (add, seeds attrs from the document), `PUT .../blocks/{blockId}` (edit key/values), `DELETE .../blocks/{blockId}`, `POST .../blocks/{blockId}/copy`, `POST .../blocks/{blockId}/run` (assemble prompt + real model call), `POST .../blocks/{blockId}/responses/{responseId}/reparse` (re-parse an existing response into document attributes). `mode.go` serves `GET /api/modes`; `model.go` serves `GET /api/models`; `attribute.go` serves `GET /api/attribute-values?key=` (the caller's distinct past values for an attribute key, for autocomplete); `label.go` serves `GET /api/labels` (the caller's label names) and `POST /api/documents/{id}/suggest-labels` (read-only: analyzes the document's `story` attribute via `llm.SuggestLabels` and returns suggested label names for the caller to apply itself via `PUT`; 400s when the document has no `story` yet), and `POST /api/documents/{id}/recommend-labels` (the same, but via `llm.SelectLabels` — returns the subset of the caller's existing labels that fit the story). Labels are user-global tags; a label is deleted once it's attached to no document (orphan cleanup in `SetDocumentLabels` and `DeleteDocument`, which is therefore transactional and takes `userID`). Shared helpers `ownedDocument`/`findBlock` enforce ownership. Conventions: resources owned by another user return **404, not 403** (don't leak existence); list endpoints guard `nil` slices so the JSON body is `[]`, never `null`
- `internal/handler/reddit.go` — optional Reddit import feature. `GET /api/reddit/posts` lists newest posts from the user's configured subreddit; `GET /api/reddit/post?url=` fetches one post. Uses an app-only OAuth token (`client_credentials` grant) cached and refreshed in a shared `redditAuth` (anonymous JSON access is blocked; Reddit requires a descriptive `User-Agent`). Both 503 when `REDDIT_CLIENT_ID`/`_SECRET` are unset. Read-only: the app-only token has no user identity and **cannot submit**. Submitting is separate — `POST /api/documents/{id}/reddit-submit` (`SubmitRedditPost`, nested so it own-checks via `ownedDocument`) publishes a self post with a `password`-grant token (the script-app account `REDDIT_USERNAME`/`REDDIT_PASSWORD`, 503 when unset), saves the returned permalink to the `document_posts` child table, and returns the refreshed document. `PUT /api/auth/me` (`UpdateMe` in `auth.go`) is a partial update (optional-pointer body) of per-user settings: `subreddit` (validated, defaults to `WritingPrompts`; migration `000004`) and `streamingEnabled` (migration `000005`; toggles live-streamed responses, surfaced as a Switch in `AccountMenu` and consumed by `BlockCard`/`api.postStream` over an NDJSON `POST .../run/stream`). Frontend: `SettingsPage`, `RedditPostsPage`, `RedditPromptDialog`/`RedditUrlDialog`, `RedditSubmitDialog` (document ⋮ menu; seeds title from the original prompt and body from the `story` attribute)
- `internal/handler/reflex.go` — the read-only **"Anansi"** browser over the legacy `reflex.db` SQLite file (an older version of this app; schema maps onto the current models: `doc`→Document, `block`→Block (carries its own `model`, surfaced on each `Response.Model`), `blockresponse`→Response, `doctag`/`blocktag`→attributes, `label`→labels). `GET /api/anansi/documents` (list, newest first, archived included) and `GET /api/anansi/documents/{id}` (full aggregate) — require a logged-in session but are **not** user-scoped (legacy data is shared and read-only). Backed by `store.ReflexStore` (`store/reflex.go`) over `*sql.DB` opened via `db.OpenSQLite` (`mode=ro&immutable=1`, driver `modernc.org/sqlite`, pure-Go); the SQLite handle is opened in `main.go` from `cfg.ReflexDBPath` (env `REFLEX_DB_PATH`, default `../reflex.db`, gitignored). The driver returns `DATETIME` columns as `time.Time` directly. Frontend: landing-page card → `AnansiPage` (`/anansi`, reuses `DocumentList`/`DocumentRow` via their optional `basePath`/`showArchived` props) → read-only `AnansiDocumentPage` (`/anansi/:id`), a thin wrapper over the shared `components/ReadOnlyDocumentPage` (also used by Charlotte). `ReadOnlyDocumentPage` deliberately mirrors the live document-page structure read-only — collapsible block `<details>` cards with mode headers, read-only key/values, and a structured/raw response toggle ported from `BlockCard`/`DocumentAttributes`, minus all edit/run/save/delete affordances; keep it in sync when those components change
- `internal/handler/charlotte.go` — a parallel read-only **"Charlotte"** browser, same shape as Anansi but over the external `charlotte-cli` executable (an older file-based version of this app). `GET /api/charlotte/documents` / `/{id}`, logged-in but not user-scoped. Backed by `store.CharlotteStore` (`store/charlotte.go`), which shells out: `charlotte-cli --list` enumerates document names (used as titles), `charlotte-cli --doc <name>` dumps one as JSON (`docTitle`/`docUrl`/`docTags`/`docBlocks[]` → the same `model.Document` aggregate; `parseCharlotteDoc` does the pure mapping). Docs are identified by name, not id — the store assigns stable ids by indexing the **sorted** `--list` output and resolves id→name again on fetch. A non-zero CLI exit (the legacy tool fails to parse a doc) surfaces as an error. Executable from `cfg.CharlotteCLI` (env `CHARLOTTE_CLI`, default `charlotte-cli` on `PATH`); unlike Anansi's SQLite, a missing CLI does **not** fail startup. Frontend: landing-page card → `CharlottePage` (`/charlotte`, `defaultSort="alpha"` + `hideTime` since the CLI exposes only names) → `CharlotteDocumentPage` (`/charlotte/:id`, wraps `ReadOnlyDocumentPage`)
- `internal/mode` — the fixed mode registry: `Mode{Name, Label, Keys, Output, Template, Tools}`, with mustache templates embedded from `templates/*.mustache` via `go:embed`. `All()` / `Get(name)`. The `Template` and `Tools` fields are `json:"-"` so they stay server-side. Add a mode by adding an entry plus its template file; attach tools by setting `Tools: []llm.Tool{...}` (see the LLM section). **Per-user template overrides**: `TemplateFor(username, mode)` (used by `RunBlock`) reads a runtime override from `<TemplatesBaseDir>-<username>/<mode>.mustache` (sibling of the default dir) when present, else falls back per-file to the embedded default. The shared system prompt works the same way via `SystemPrompt(username, provider)` (per-user override file `<TemplatesBaseDir>-<username>/system.mustache`, default `templates/system.mustache`), but first prefers a per-provider override `system-<provider>.mustache` (provider = the selected model's `llm.ProviderFor(id)`, e.g. `anthropic`/`openai`) when present, so a user can tune the system prompt per provider; all go through the internal `override`/`lookupOverride` helpers. `TemplatesBaseDir` is set from `cfg.ModeTemplatesDir` (env `MODE_TEMPLATES_DIR`, default `internal/mode/templates`). Usernames are validated (`[A-Za-z0-9_-]` only) to prevent path traversal; the override dirs (`templates-*/`) are git-ignored
- Response processing lives in `internal/handler/response.go`: `parseTopLevelTags` byte-scans free-form responses (not `encoding/xml`) for top-level tags, counting same-name nesting depth and degrading gracefully on malformed input (an unclosed final tag is auto-closed at end of string). The frontend's `parseResponseSegments` (`frontend/src/lib/responseSegments.ts`) mirrors this scan for ordered rendering and must be kept in sync. `RunBlock` merges the parsed tags (plus the `output` key when set) via `MergeDocumentAttributes`. Covered by `response_test.go` (run with `make backend-test`; see also `internal/llm/names_test.go`)
- `internal/llm` — provider-agnostic LLM wrapper over the [GoAI SDK](https://github.com/zendev-sh/goai). Holds the fixed cross-provider model list (`Models()` / `Valid(id)`) and `Generate(ctx, model, system, prompt, tools)` (the system prompt is supplied by the caller — see `mode.SystemPrompt`). Each `Model`'s `Provider` field selects the GoAI provider client (`clientFor` routes to `anthropic`/`openai`/`google`/`openrouter`/`deepseek` `.Chat`), so a model's `ID` is the name that client expects (a vendor-native id, or an OpenRouter slug like `z-ai/glm-5.2`). Aliases GoAI's `Tool` type so modes attach tools without importing the vendor library. The only place vendor/provider code lives; add a model by appending to the list with its native ID and provider. `GenerateStream(..., onDelta)` is the streaming sibling of `Generate`: it consumes GoAI's raw chunk stream (`ts.Stream()`) so reasoning streams live, frames it like `combineSteps` (`<thinking>`/`<toolname>` tags), and returns the same final value via `combineSteps(ts.Result())`. GoAI gotchas (v0.8.5): `ts.TextStream()` forwards only text (no reasoning); `WithOnStepFinish` fires at step *end* (`StepResult.ToolResults` is always empty there); `ChunkToolResult` is never emitted — tool results land only in the final result; the agentic tool loop runs automatically however chunks are consumed
- `internal/model` — plain domain structs mirroring the DB schema (no data-access logic); JSON-tagged, aggregate-shaped for API bodies
- `internal/store` — `Store` wraps the pool with raw-SQL CRUD methods over the models; returns `store.ErrNotFound` for missing rows. `GetDocument` loads the full aggregate (blocks → attributes/responses) with batched queries
- **Empty Go slices marshal to JSON `null`, not `[]`** — embedded aggregate slices (`Document.Blocks`/`Labels`, `Block.Responses`) must be defaulted to an empty slice in the store, or the frontend crashes on `.length`/`.map`. `GetDocument`/`GetBlock` guard `Blocks`/`Responses`; `Labels` is not yet guarded, so guard slice access with `?? []` on the frontend too

Routing uses `go-chi/chi`. Handlers are plain `http.HandlerFunc` (no framework-specific types). CORS is handled by `rs/cors` middleware — it's unused during local dev (covered by the Vite proxy) but activates in production.

**Database** (`db/migrations/`) — plain SQL files managed by `golang-migrate`. Naming convention: `000001_<name>.up.sql` / `000001_<name>.down.sql`. Local credentials: `nisaba/nisaba/nisaba` (user/password/db).
If `make migrate` reports "no change" but a column is missing, the dev DB's `schema_migrations` version has drifted ahead of the actual schema — recover with `migrate force <version>` then `make migrate`. Domain tables use `BIGSERIAL` ids and `ON DELETE CASCADE` FKs. String key/value attributes live in normalized child tables with `PRIMARY KEY (parent_id, key)` for uniqueness; free-form `metadata` is a `JSONB` column.

## Adding a New API Endpoint

1. Add a handler function in `backend/internal/handler/` returning `http.HandlerFunc`
2. For user-scoped data, read the caller via `sess.UserID(r)` (401 if absent) and scope/own-check on it — see `document.go`
3. For resources nested under a document (`/documents/{id}/...`), reuse the `ownedDocument`/`findBlock` helpers in `block.go` and register inside the nested `r.Route("/{id}", ...)` group rather than re-checking ownership
4. Register the route in `backend/cmd/server/main.go` inside the `/api` route group
5. Call it from `frontend/src/` using a relative `/api/...` path
